## About the Summary Field

Each paper includes a short **Summary** shown in the detail modal. These summaries are **generated by a large language model (LLM)** based on the paper's title and abstract, condensed into an IMRaD-style overview (Introduction, Methods, Results, and Discussion).

> âš ï¸ **Summaries are AI-generated and for reference only.** They may contain inaccuracies or miss important nuances. Always refer to the original paper for authoritative content.

## Writing Reading Notes

Reading notes are plain Markdown files stored in the `learning/` folder. The filename must match the paper's ID.

### Finding the Paper ID

Each paper has a short ID in the format `{first-author-surname}-{year}`, e.g. `chang-2025`. You can find it:
- On each card in Learning Mode (shown in the top-right corner)
- In the detail modal
- In the downloaded JSON (`"id"` field)

### Creating a Note

Create a file named `learning/{paper-id}.md`. For example, for the paper with ID `chang-2025`:

```
learning/chang-2025.md
```

Write your notes in Markdown. There is no required format â€” use whatever structure works for you. Here is a suggested template:

```markdown
# Key Findings

- Main result 1
- Main result 2

## Methods
- Method or technique used
- Analysis approach

## My Thoughts
- Questions or ideas for follow-up
- Connections to other papers
```

## Sample `papers.json`

A sample `learning/papers.json` is included with one entry (`chang-2025`) so you can see the expected format. Replace or extend it with your own selection from the Download button.

The format is a JSON array of paper objects:

```json
[
  {
    "id": "chang-2025",
    "title": "Unraveling landslide failure mechanisms with seismic signal analysis...",
    "authors": ["Juiâ€Ming Chang", "Cheâ€Ming Yang", "..."],
    "year": 2025,
    "category": "landslide",
    "journal": "Natural hazards and earth system sciences",
    "doi": "",
    "url": "",
    "summary": "This study likely explores landslide failure mechanisms..."
  }
]
```

Required fields: `id`, `title`. All other fields are optional but recommended for a good display.
